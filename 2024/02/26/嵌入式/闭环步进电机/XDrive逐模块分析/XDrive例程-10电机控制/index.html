代码中涉及后续模块的部分，可以参考后续对应章节。

功能实现

void Control_PID_To_Electric(int32_t _speed)

  • 功能：PID电流控制

  • 参数：

    • _speed 控制速度

  • 返回：

  • 实现：

    • 误差处理

      • 保存上一次速度误差v_error_last

      • 计算本次速度误差到v_error

        本次误差计算用到了当前速度（输入参数）-估计速度（储存于控制器对象的est_speed变量内）

      • 误差限幅

        将误差限制在

    • 计算PID

      • 计算比例部分（op）

      • 计算积分部分（oi）

        以1024为界，分为两部分，除以1024，整数部分用于计算输出，余数部分用于下一次积分计算具体公式如下

        • 计算结束后限幅，限制为额定电流 * 1024
          $$
          [-额定电流1024,额定电流1024]
          $$

      • 计算差分部分（od）

      • 累加各部分计算值，然后输出

      • 总输出限幅，限制在额定电流

    • 输出FOC电流

      输出电流大小等于PID输出

    • 输出FOC位置

      • 电流大于0，则在现在位置基础上加一个步进细分(256)
      • 电流小于0，则在现在位置基础上减一个步进细分(256)

    • 输出任务到驱动REIN_HW_Elec_SetDivideElec()

功能实现

void Control_DCE_To_Electric(int32_t _location, int32_t _speed)
  • 功能：DCE电流控制，让电机动起来
  • 参数：

    • _location：控制位置

    • _speed：控制速度

  • 返回：
  • 实现：

    • 误差处理

      • 计算位置误差p_error

        本次误差计算用到了当前位置（输入参数）-估计速度（储存于控制器对象的est_location变量内）

      • 计算速度误差v_error，并缩小128倍

        本次误差计算用到了当前速度（输入参数）-估计速度（储存于控制器对象的est_speed变量内）

      • 误差限幅

        将位置误差限制在1/16圈内(51200/16)

        将速度误差限制在10r/s内(51200*10/128)

    • 计算PID

      • 计算比例部分（op），依靠位置误差

      • 计算积分部分（oi）

        以1024为界，分为两部分，除以1024，整数部分用于计算输出，余数部分用于下一次积分计算具体公式如下

        • 计算结束后限幅，限制为额定电流 * 1024
          $$
          [-额定电流1024,额定电流1024]
          $$

      • 计算差分部分（od）

      • 累加各部分计算值，然后输出

      • 总输出限幅，限制在额定电流

    • 输出FOC电流

      输出电流大小等于PID输出

    • 输出FOC位置

      • 电流大于0，则在现在位置基础上加一个步进细分(256)
      • 电流小于0，则在现在位置基础上减一个步进细分(256)

    • 输出任务到驱动REIN_HW_Elec_SetDivideElec()

void Motor_Control_Callback(void)

  • 功能：控制器任务回调

  • 参数：

  • 返回：

  • 实现：

    • 首次进入控制回调

      • 读取(为了方便将XDrive代码移植到软件编码器,将位置读取初始化部分全部放置在此处运行)

        将编码器全局变量内的读数值，赋值给控制全局变量里的四个值

        real_lap_location 、real_lap_location_last、 real_location 、real_location_last

      • 第一次运行强制退出本次中断

    • 数据采集

      • 读取单圈位置
        • 保存上一次的位置到real_lap_location_last
        • 从mt6816.rectify_angle中获取本次的位置到real_lap_location
      • 整圈检测，修正增量值
        • 计算差值增量，用于检测是否发生了圈数变化，算入变量sub_data中
      • 读取位置
        • 寄存上一次的全局位置real_location_last
        • sub_data中的值累加到全局位置中，更新本次的全局位置real_location

    • 数据估计

      • 估计速度
        $$
        est_speed^{(n+1)}=\frac{\frac{\Delta P}{T}+31est_speed^{(n)}}{32} \
        est_speed_mut=(\frac{\Delta P}{T}+31        est_speed^{(n)})%32
        $$

      • 估计位置

        • 超前位置est_lead_location由速度补偿函数Motor_Control_AdvanceCompen()根据估计速度算出
        • 估计位置est_location等于实际读取位置real_location累加超前位置est_lead_location

      • 估计误差

        估计误差est_error等于软位置soft_location减去估计位置est_location

    • 运动控制

      • 输出休眠(禁用了输出IO,恢复输出需要单独处理)

        进入条件：堵转标志置位 或 软目标_失能指令 或 编码器校准表无效

        • 清除积分
        • 清FOC位置foc_location
        • 清FOC电流foc_current
        • 驱动休眠

      • 输出刹车

        进入条件：软目标_刹车指令

        • 清除积分
        • 清FOC位置foc_location
        • 清FOC电流foc_current
        • 驱动休眠

      • 根据运动模式，实施到带电机控制

        • 停止模式Control_Mode_Stop：电机休眠

        DIG控制中的模式（由CAN/RS485触发）

        • 位置控制模式Motor_Mode_Digital_Location：调用DCE控制函数Control_DCE_To_Electric()，输入软位置soft_location、软速度soft_speed
        • 速度控制模式Motor_Mode_Digital_Speed：调用PID控制函数Control_PID_To_Electric()，输入软速度soft_speed
        • 电流控制模式Motor_Mode_Digital_Current：调用电流控制函数Control_Cur_To_Electric()，输入软电流soft_current
        • 轨迹控制模式Motor_Mode_Digital_Track：调用DCE控制函数Control_DCE_To_Electric()，输入软位置soft_location、软速度soft_speed（240507ps：可能与位置控制模式的差别体现在后续计算的软位置的软速度上）

        MoreIO控制中的模式（由PWM/PUL触发）

        • 位置控制模式Motor_Mode_PWM_Location：调用DCE控制函数Control_DCE_To_Electric()，输入软位置soft_location、软速度soft_speed
        • 速度控制模式Motor_Mode_PWM_Speed：调用PID控制函数Control_PID_To_Electric()，输入软速度soft_speed
        • 电流控制模式Motor_Mode_PWM_Current：调用电流控制函数Control_Cur_To_Electric()，输入软电流soft_current
        • 轨迹控制模式Motor_Mode_PULSE_Location：调用DCE控制函数Control_DCE_To_Electric()，输入软位置soft_location、软速度soft_speed（240507ps：可能与位置控制模式的差别体现在后续计算的软位置的软速度上）

    • 模式变更

      笔者的理解中，这个mode_order的功能就像一个影子寄存器，它的作用是使控制模式不会立即改变，而是放在了下一次循环中改变。这里的功能主要是对Signal_MoreIO的处理，在DIG启用时，使之关闭，当使用Signal_MoreIO时，使其配置到对应模式。当mode_order中的值发生变更时，进入以下程序片段：

      • 更新控制器运行模式mode_run为新的mode_order
      • Control_Mode_Stop：关闭Signal_MoreIO_Config(MoreIO_Mode_Disable);、创建新曲线soft_new_curve = true

      DIG(CAN/RS485)

      • 关闭Signal_MoreIO_Config(MoreIO_Mode_Disable);、创建新曲线soft_new_curve = true

      MoreIO(PWM/PUL)

      • Motor_Mode_PWM_Location：Signal_MoreIO_Config(MoreIO_Mode_PWM_Location)、创建新曲线soft_new_curve = true
      • Motor_Mode_PWM_Speed：Signal_MoreIO_Config(MoreIO_Mode_PWM_Speed)、创建新曲线soft_new_curve = true``
      • Motor_Mode_PWM_Current：Signal_MoreIO_Config(MoreIO_Mode_PWM_Current)、创建新曲线soft_new_curve = true
      • Motor_Mode_PULSE_Location：Signal_MoreIO_Config(MoreIO_Mode_PULSE_Locatioon)、创建新曲线soft_new_curve = true

    • 硬目标提取

      这一部分主要服务于MoreIO部分的目标提取，因为DIG控制中硬目标是由通讯协议直接写入控制器的。而MoreIO时同时PWM/PUL来读取目标的。

          • 脉冲位置跟随

          • 清除堵转检测

          DIG(CAN/RS485)所属模式：

          • Motor_Mode_Digital_Speed：速度追踪器创建一个新任务Speed_Tracker_NewTask()，输入为：估计速度est_speed
          • Motor_Mode_Digital_Current：电流追踪器创建一个新任务Current_Tracker_NewTask()，输入为：foc电流foc_current

          MoreIO(PWM/PUL)所属模式：

          • Motor_Mode_PWM_Location：位置追踪器创建一个新任务Location_Tracker_NewTask()，输入为：估计位置est_location、估计速度est_speed
          • Motor_Mode_PWM_Speed：速度追踪器创建一个新任务Speed_Tracker_NewTask()，输入为：估计速度est_speed
          • Motor_Mode_PWM_Current：电流追踪器创建一个新任务Current_Tracker_NewTask()，输入为：foc电流foc_current
          • Motor_Mode_PULSE_Location：位置插补器创建一个新任务Move_Reconstruct_NewTask()，输入为：估计位置est_location、估计速度est_speed

            • 从电流追踪器变量go_current读出软速度soft_current

            • 从位置追踪器变量go_location读出软位置soft_location

    • 状态识别

      识别堵转或过载，其本质是对电流大小的检测

      • 暂存电流foc_current的绝对值到变量abs_out_electric中

      • 堵转检测

        • 当工作在电流模式下、有输出电流值、估计速度低于1/5转/s时
          • 持续时间超过1s，则堵转标志位置true
          • 否则，堵转时间累计器持续累加堵转时长
        • 当工作在其他模式下，电流值处于最大值3000且转速低于1/5转/s时
          • 持续时间超过1s，则堵转标志位置true
          • 否则，堵转时间累计器持续累加堵转时长
        • 其他状态时，持续清零堵转时间累计器。

      • 过载检测

        • 当电流值等于最大电流值3000

          • 当电流值持续等于最大电流值3000超过1秒时
            • 堵转标志位置true
            • 过载时间累计器持续累加每次中断的周期数

        • 否则

          清零过载时间累计器

          过载标志置false

    • 状态记录

      这一部分用来检测当前的系统运行状态

      • mode运行在停止模式

        此时电机的运行状态state为停止Control_State_Stop

      • 堵转标志置位为true

        此时电机的运行状态state为堵转Control_State_Stall

      • 过载标志置位为true

        此时电机的运行状态state为过载Control_State_Stall

      • 运行在DIG的位置模式下时Motor_Mode_Digital_Location

        • 当软位置等于目标位置，且软速度为0时

          电机的运行状态均为“软硬目标匹配”Control_State_Finish

        • 否则

          此时电机的运行状态state为运行中Control_State_Running

      • 运行在DIG的位置模式下时Motor_Mode_Digital_Location

        • 当软速度等于目标速度时

          电机的运行状态均为“软硬目标匹配”Control_State_Finish

        • 否则

          此时电机的运行状态state为运行中Control_State_Running

      • 运行在DIG的位置模式下时Motor_Mode_Digital_Location

        • 当软电流等于目标电流时

          电机的运行状态均为“软硬目标匹配”Control_State_Finish

        • 否则

          此时电机的运行状态state为运行中Control_State_Running

      • 运行在其他模式下时

        电机的运行状态均为“软硬目标匹配”Control_State_Finish
